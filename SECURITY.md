# Security Policy

## Supported Versions

Security fixes are prioritized for the latest supported version of Chaemera.

If you are running an older build, please reproduce the issue on the latest available release before reporting it.

## Reporting a Vulnerability

Please report suspected security vulnerabilities through GitHub Security Advisories for this repository rather than filing a public issue.

Please avoid posting exploit details, secrets, tokens, private URLs, or reproduction steps for a live vulnerability in public issues or discussions.

When possible, include:

- affected version or commit
- platform and environment details
- impact summary
- reproduction steps
- any suggested mitigation or workaround

If the GitHub advisory flow is unavailable for any reason, open a minimal private contact path through the repository owner and avoid posting sensitive details publicly.
