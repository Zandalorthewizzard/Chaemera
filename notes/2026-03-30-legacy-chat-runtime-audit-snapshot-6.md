## Key Separation That Now Matters

- Live-proven runtime/core on current branch:
  - packaged/debug Tauri entry into real chat runtime
  - home composer -> create app -> stream
  - direct `app-details` route -> `Open in Chat` -> stream
  - import-app core path with existing `AI_RULES.md`
  - copy-message behavior after streamed output
- Remaining partial rows are mostly not transport-collapse anymore.
- They are a mix of:
  - legacy runtime design debt,
  - adapter omissions on the worker path,
  - and metadata/cancel semantic gaps.

## Manual Verification Bucket For Final EXE Pass

These are the checks most worth deferring to the user's final live `.exe` walkthrough because they are either shell-sensitive, provider-sensitive, or expensive to prove cleanly in WebDriver:

1. model/rules/settings sensitivity actually changes the answer in a visible way
2. text attachment, image attachment, and selected component context in one real session
3. cancel during an active stream and, if possible, during a repair-like follow-up pass
4. redo on a transcript that already contains previous assistant output and code changes
5. file-targeted question that proves codebase extraction is materially helping
6. any native dialog/focus/modal flows, especially `Import App -> Select Folder`

## Current Audit Interpretation

- The current branch is no longer dominated by `blocked-by-transport` chat failures.
- The main residual defects are now narrower:
  - double-terminal end/error behavior in the Rust host
  - incomplete completion metadata forwarding
  - partial cancel-state reconciliation
  - dormant MCP/local-agent hooks on the worker path
- This narrows the eventual Codex daemon replacement scope discussion significantly:
  - core stream/persistence path is real and partially healthy,
  - but the richer legacy behaviors are fragmented enough that they should not be treated as a clean foundation.
