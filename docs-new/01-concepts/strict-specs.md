# Strict Task Specifications

Every implementation task must define:

1. Goal and non-goals.
2. Locked decisions and ownership boundaries.
3. Ordered implementation tasks.
4. Requirement to test to gate mapping.
5. Rollback procedure.

Canonical docs without evidence are invalid and must be marked `UNKNOWN`.
